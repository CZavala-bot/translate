# 04-workflows

Flujos de trabajo prácticos para usar el framework en el día a día.

## Objetivo
Reducir fricción y evitar dudas sobre:
- qué documentar
- cuándo hacerlo
- en qué formato

## Regla
Si una duda se repite, se convierte en workflow.

## Contenido
- `day-to-day.md`: cuándo crear FR/NFR/ADR/entregables
- `how-to-ingest-sources-docling.md`: ingestión de fuentes con Docling
- `how-to-create-fr.md`: crear un FR consistente
- `how-to-create-adr.md`: registrar una decisión (ADR)
- `how-to-publish-deliverable.md`: publicar un entregable versionado
