# curated

Conocimiento destilado y estructurado.

## Qué va aquí
- requisitos funcionales (FR)
- requisitos no funcionales (NFR)
- decisiones (ADR)
- historias de usuario
- análisis
- diseños

## Regla
Todo documento aquí debe:
- tener `id`, `owner`, `status` y `date`
- incluir `sources:` con al menos 1 referencia (archivo en `sources/raw/` y/o URL externa)
- si se apoya en `ingested/`, se puede mencionar en el cuerpo, sin reemplazar la fuente original
